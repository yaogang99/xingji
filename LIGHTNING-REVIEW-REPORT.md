## 🔴 严重问题 (必须修复)

### 问题 #1: 数据系统 - 资源产出公式缺失
**位置**: 01-data-system.md, 14-production-recipes.md  
**问题**: 设施产出公式只定义了产出数量，但未定义产出频率/周期

**当前定义**:
```typescript
// 只有产出数量，没有时间
outputs: [{ resourceId: 'iron_ore', amount: 1 }]
```

**缺失内容**:
```typescript
// 需要补充产出频率
productionCycle: number;  // 产出周期(秒)
productionRate: number;   // 每分钟产出 = amount / cycle * 60
```

**影响**: 无法计算实际产出速度，离线收益算法无法正确实现

**修复建议**:
- 为所有33种设施补充`productionCycle`字段
- 在设施配方表中增加产出频率列

---

### 问题 #2: 核心系统 - 离线收益算法冲突
**位置**: 02-core-system.md (Step 5算法) vs 07-balance-design.md  
**问题**: 两处离线收益算法定义不一致

**02-core-system.md 定义**:
```typescript
// 72小时以上统一50%
if (hours > 24) {
    rate = 0.5;
    capped = hours > 72;
}
```

**07-balance-design.md 定义**:
```typescript
// 24-72小时80%，72+小时50%
if (hours <= 24) {
    rate = 0.8;
} else {
    rate = 0.5;
}
```

**冲突点**: 24-72小时区间定义不一致（50% vs 80%）

**修复建议**: 统一采用三段式：
- 0-8小时: 100%
- 8-24小时: 80%
- 24-72小时: 50%

---

### 问题 #3: 数值平衡 - 生产链利润矛盾
**位置**: 07-balance-design.md (资源利润表)  
**问题**: 电路板利润率-23%，但文档说"鼓励玩家优化产业链"

**当前定义**:
| 产品 | 利润率 |
|------|--------|
| 电路板 | -23% |
| AI芯片 | -53% |

**问题**: 负利润率的产品无法通过优化产业链变成正利润

**修复建议**:
- 明确负利润产品的设计意图（仅用于贸易/探险）
- 或调整基础数值使高级产品有合理利润空间
