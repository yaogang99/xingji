### PATH-001: 资源路径使用绝对路径

**发现场景**: 用户级测试 - 黑屏问题  
**问题描述**: `index.html` 中使用 `/assets/xxx` 绝对路径，导致直接打开文件时404  
**影响**: 玩家双击打开游戏时黑屏，无法运行  

**设计修改**:
```diff
- <script src="/assets/index.js">
+ <script src="./assets/index.js">
```

**解决方案**:
1. 构建后自动修复路径
2. 添加 `npm run fix-paths` 命令
3. 每次构建自动执行

**实现**:
```json
"scripts": {
  "build": "tsc && vite build && npm run fix-paths",
  "fix-paths": "sed -i '' 's|src=\"/assets/|src=\"./assets/|g' dist/index.html"
}
```

**验证标准**:
- [x] 双击打开能正常运行
- [x] 使用HTTP服务器也能运行

### UI-004: 主菜单逻辑错误

**发现时间**: 2026-03-17 12:01  
**发现场景**: 用户反馈  
**问题描述**: 主菜单逻辑不符合标准游戏模式：
- 无存档时只有一个"开始游戏"按钮
- 有存档时显示"继续游戏"+"新游戏"
- 没有独立的"读取存档"选项
- "开始游戏"和"新游戏"概念混淆

**影响**: 玩家无法明确选择"新游戏"还是"读取存档"，体验不符合游戏行业标准

**正确逻辑**:
```
主菜单
├── 🎮 新游戏 → 选择档位（空档位直接开始，有存档提示覆盖）
├── 💾 读取存档 → 选择档位（只能选有存档的档位）
├── ⚙️ 设置
└── ℹ️ 关于
```

**设计修改**:
```diff
- 无存档时: [开始游戏]
- 有存档时: [继续游戏] [新游戏]

+ 始终显示: [新游戏] [读取存档] [设置] [关于]
```

**实现文件**:
- `src/ui/MainMenu.tsx` - 重写按钮逻辑
- `src/ui/SaveSlotPanel.tsx` - 添加 'new' 模式支持
- `src/App.tsx` - 区分 'select-slot-new' 和 'select-slot-load' 视图

**技术变更**:
```typescript
// 新增视图类型
type ViewType = 'menu' | 'select-slot-new' | 'select-slot-load' | ...

// MainMenu Props 变更
- onStartGame: () => void
+ onNewGame: () => void

// SaveSlotPanel 新增模式
- mode: 'load' | 'save'
+ mode: 'load' | 'save' | 'new'
```

**验证标准**:
- [x] 主菜单始终显示"新游戏"和"读取存档"按钮
- [x] 点击"新游戏"进入选择档位界面（可覆盖已有存档）
- [x] 点击"读取存档"进入选择档位界面（只能读取已有存档）
- [x] 选择空档位开始新游戏
- [x] 选择有存档档位时提示是否覆盖（新游戏模式）或直接读取（读取模式）

**状态**: ✅ 完成

### UI-005: 新游戏不应先选档位

**发现时间**: 2026-03-17 12:08  
**发现场景**: 用户反馈  
**问题描述**: 
- UI-004 修复后，点击"新游戏"仍需要先选择档位
- 这不符合标准游戏逻辑：新游戏应该直接开始，存档是之后的事
- 玩家想"开始新游戏"时，不应该被询问档位问题

**影响**: 用户体验不符合行业标准，新游戏流程被中断

**正确逻辑**:
```
新游戏
    ↓
直接开始游戏（创建新游戏状态）
    ↓
游戏过程中自动保存到默认档位，或
玩家主动保存时选择档位
```

**设计修改**:
```diff
 点击"新游戏":
-   ↓
- 选择档位（空档位才能开始）
-   ↓
- 开始游戏

+   ↓
+ 直接开始游戏
+   ↓
+ 游戏中保存时选择档位
```

**实现文件**:
- `src/App.tsx` - 修改 `startNewGame` 函数，不再传入档位
- `src/App.tsx` - 删除 `select-slot-new` 视图
- `src/ui/MainMenu.tsx` - 保持不变（已正确）
- `src/ui/SaveSlotPanel.tsx` - 删除 `mode='new'`（不再需要）

**技术变更**:
```typescript
// startNewGame 简化
const startNewGame = useCallback(() => {
  setCredits(1000)
  setResearchPoints(0)
  setResources(INITIAL_RESOURCES)
  setPlayTime(0)
  setCurrentSlotIndex(null)  // 新游戏没有当前档位
  setGameStarted(true)
  setView('main')
}, [])

// 删除视图类型
type ViewType = 'menu' | 'select-slot-load' | ...  // 删除 'select-slot-new'

// 删除模式类型
type SlotSelectMode = 'load'  // 删除 'new'
```

**验证标准**:
- [x] 点击"新游戏"直接进入游戏主界面
- [x] 新游戏创建全新状态（1000货币，0资源等）
- [x] 不需要先选择档位
- [x] 读取存档仍需选择档位

**状态**: ✅ 完成
