### Level 2: E2E自动化测试（新增强制要求）

**不仅仅是"写配置"，必须实际运行**

| 检查项 | 要求 | 验证方法 |
|--------|------|----------|
| E2E运行 | 必须实际运行 | `npm run test:e2e` |
| 截图验证 | 每个场景有截图 | 检查 `test-results/` |

**v1.0 错误**: 只写了配置，没要求运行  
**v2.0 修正**: 必须实际运行并截图验证

### Level 3: 用户层测试（新增强制要求）⭐

**必须像真实用户一样操作**

| 检查项 | 要求 | 验证方法 |
|--------|------|----------|
| **真实打开** | 双击打开HTML文件 | 不是开发服务器 |
| **黑屏检测** | 加载动画后看到内容 | 肉眼验证 |
| **按钮点击** | 所有导航按钮可点击 | 实际操作 |
| **数据展示** | 资源/货币正常显示 | 肉眼验证 |
| **交互响应** | 出售/研究/探险可用 | 实际操作 |
| **Network检查** | F12无404错误 | 开发者工具 |
| **Console检查** | 无红色报错 | 开发者工具 |

## 教训

> **"代码正确" ≠ "可运行" ≠ "用户能用"**

航天级必须验证：
1. **代码层** - 单元测试
2. **自动化层** - E2E测试  
3. **用户层** - 真实操作 ⭐**缺失**

**v2.0 核心改进**: 用户层测试从"建议"变为"强制"。
